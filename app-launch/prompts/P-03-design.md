# P-03 | 生成设计需求文档

**角色**：资深产品设计负责人
**输入依赖**：`CLAUDE.md` + `docs/app-launch/01-framework.md`
**输出文件**：`docs/app-launch/03-design-brief.md`

```
你是一个资深产品设计负责人，负责向 UI 设计师交付可直接执行的设计需求文档。

请读取 CLAUDE.md 和 docs/app-launch/01-framework.md，生成设计需求文档，输出为 docs/app-launch/03-design-brief.md。
设计师应能不需要额外沟通，直接基于此文档开始 Figma 设计。

文档必须包含：
1. 页面清单（编号 / 页面名称 / 功能说明 / 关键 UI 元素 / 交互要点）
2. 设计风格规范
   - 主色调 + 辅色（给出色值参考范围）
   - 字体方案（标题/正文/数字/标签）
   - 视觉密度（轻量/标准/信息密集）
   - 图标风格（线性/面性/混合）
3. 竞品风格差异对比表（参考竞品 / 相似之处 / 本项目差异点 / 原因）
4. 公共组件体系（按钮 / 弹窗 / 卡片 / 表单 / 加载态 / 空状态）
5. 应用商店视觉素材清单（截图数量 / 尺寸要求 / 内容主题）
6. 四周设计排期（每周交付哪些页面/组件，与 01-framework.md 对齐）
7. 交付规范（切图命名 / 标注工具 / 文件格式 / 提交方式）
8. 设计注意事项（品牌禁忌 / 合规展示要求 / 特殊机型适配）

格式要求：
- 页面清单用表格，按功能模块分组（如：首页模块 / 借款流程 / 个人中心）
- 总页面数量与 CLAUDE.md 中描述的业务范围匹配
```

**验收标准**：
- [ ] 页面清单覆盖完整用户旅程（从首次打开到核心业务完成）
- [ ] 色值方案与竞品有明显差异（不能和竞品撞色）
- [ ] 四周排期可执行（工作量分布合理，不头重脚轻）
