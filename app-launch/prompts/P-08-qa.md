# P-08 | 质检所有输出文档

**角色**：项目质检
**输入依赖**：`CLAUDE.md` + `docs/app-launch/` 下所有已生成文档（01~07）
**输出文件**：`docs/app-launch/00-index.md`（含质检报告）
**使用时机**：所有文档生成完毕后执行

```
请执行以下质检任务，并将结果输出为 docs/app-launch/00-index.md：

【第一部分：文档索引】
列出所有已生成文档（01~07），每个文档包含：
- 文件名
- 文档标题
- 核心内容摘要（3句话以内）
- 字数统计
- 最后更新时间

【第二部分：一致性检查】
对照 CLAUDE.md 的原始需求，逐项检查：

1. 产品定位是否在所有文档中保持一致？
2. 上线时间线是否在各文档（框架/设计排期/外部依赖/商店策略）中对齐？
3. 外部依赖清单：CLAUDE.md 中列出的依赖是否全部出现在 06-external-dependencies.md？
4. 竞品名称：CLAUDE.md 中的竞品是否全部出现在 02-competitor-report.md 和 03-design-brief.md？
5. 技术三方服务：04-tech-spec.md 中的三方集成是否全部出现在 05-compliance-checklist.md？

【第三部分：差异报告】
对于上述检查中发现的不一致项，列出：
- 不一致描述
- 涉及文档
- 建议修正方式

【第四部分：待人工确认项】
列出所有文档中出现的"待定/TBD/需确认"标记，汇总成一张表，
供 PM 在第一次团队对齐会上逐一确认。
```

**验收标准**：
- [ ] 差异报告为空，或每条差异都有明确修正建议
- [ ] 待人工确认项不超过 10 条（超过说明 CLAUDE.md 质量需提升）
